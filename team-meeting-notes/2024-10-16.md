## Security & Privacy Team Input
- Alan explained concerns that users have about data usage - who, what, where and why is data being stored
- What can we do to help with Umbraco Packages being transparent about their security and data retention
- Good examples:
	- Joomla "Extension Capabilities" for reporting across all extensions
	- Apple "Nutrition Labels" and privacy manifests
- An initial improvement could be a Privacy section in the Package information in the marketplace
    - Encourage best practise of adding this, not requiring it
    - Add to the meta data of a package in umbraco-marketplace.json, in as 'structured' a way as possible
- Most important for closed source products, so HQ can lead by example and do this for HQ commerical packages
- Would really help with trust with bigger organisations. But don't want to put off package developers, especially not 'hobbyist' ones

Tasks with Alan and the Security Team:
- Review the instructions on how to add meta data to a package (https://docs.umbraco.com/umbraco-dxp/marketplace/listing-your-package)
- Raise an issue on the Marketplace issue tracker (https://github.com/umbraco/Umbraco.Marketplace.Issues) with suggestions of the structure of data to store
- Then package team can review and provide input
